# 원시값 포장
- primitive 타입을 그대로 사용하지 않고, 객체로 사용하기 위해 하나의 클래스를 선언해 원시 타입의 값을 의미 있는 객체로 포장하는 것.
- 스스로 상태를 관리하게 되어 책임을 분리할 수 있다.
- 모든 원시값과 문자열을 포장한다.
    - 원시 타입?
        - 문자
        - 다양한 산술 정밀도를 갖는 정수
        - 부동소수점수
        - 참, 거짓 -> boolean
        - 참조
    - 사용하는 이유
        - 예외처리를 자신 스스로 할 수 있게 책임을 부여 -> 유지 보수를 위해!
        - 갹체에게 자율성을 제공할 수 있다.
    - 단점
        - 성능이 저하되고 불필요한 객체가 생성된다.
        - 원시값을 객체로 포장하면 메모리 사용량이 늘어난다.
